# 截图控制台功能

## 功能说明

截图成功后，图片路径会自动输出到控制台，并且可以点击路径直接打开图片。

## 实现内容

### 1. 控制台日志增强
- ✅ 添加了 `path` 字段用于存储文件路径
- ✅ 新增 `screenshot` 日志类型（绿色显示）
- ✅ 截图成功时自动展开控制台

### 2. 路径显示
- ✅ 路径以蓝色下划线链接形式显示
- ✅ 鼠标悬停时颜色变深
- ✅ 路径可以自动换行（`word-break: break-all`）

### 3. 点击打开功能
- ✅ 点击路径使用系统默认程序打开图片
- ✅ 使用 Tauri 的 `shell` 插件打开文件
- ✅ 打开失败时显示错误提示

## 使用方法

### 截图操作
1. 启动一个模拟器
2. 点击模拟器卡片上的"截图"按钮
3. 截图成功后：
   - 顶部显示成功消息
   - 控制台自动展开
   - 显示截图保存路径

### 打开图片
1. 在控制台中找到截图日志
2. 点击蓝色的文件路径链接
3. 系统默认图片查看器会打开图片

## 控制台日志格式

```
[时间] 截图已保存
       C:\Users\用户名\Pictures\screenshot_emulator-5554_20260203_143022.png
       ↑ 可点击的链接
```

## 样式说明

### 日志类型颜色
- **error** (错误): 红色 `#d32f2f`
- **success** (成功): 绿色 `#388e3c`
- **info** (信息): 蓝色 `#1976d2`
- **screenshot** (截图): 绿色 `#388e3c`

### 路径链接样式
- 默认: 蓝色 `#1976d2` + 下划线
- 悬停: 深蓝色 `#1565c0` + 无下划线
- 光标: 手型指针

## 技术实现

### 数据结构
```typescript
type ConsoleLog = {
  type: string      // 日志类型
  message: string   // 日志消息
  time: string      // 时间戳
  path?: string     // 可选的文件路径
}
```

### 打开文件
```typescript
import { open } from '@tauri-apps/plugin-shell'
await open(path)  // 使用系统默认程序打开
```

## 注意事项

- 路径必须是有效的文件系统路径
- 需要 `tauri-plugin-shell` 插件支持
- 如果文件不存在或无法打开，会显示错误提示
- 控制台在截图成功时会自动展开（即使之前是折叠状态）
